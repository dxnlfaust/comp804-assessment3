Name

Web Attacks

SQL Injection

Source: https://www.avast.com

 Structured Query Language (SQL) Injection is a coding method employed to alter or extract data from SQL databases. By inserting specific SQL statements into an input field, malicious actors can run commands enabling data retrieval, sensitive information compromise, or other manipulative actions within the database. This can include gaining administrator-level privileges, or gaining root access to a machine.

Source: www.cloudflare.com

Cross-site scripting (XSS)

Source: https://www.cloudflare.com

 Cross-site scripting (XSS) is a security exploit allowing attackers to inject malicious code into a website, executed by unsuspecting users. It lets attackers bypass access controls, steal sensitive information, and manipulate content. XSS ranked as the seventh most common web app vulnerability in 2017. Successful attacks result from inadequate validation, enabling malicious scripts to access cookies, session tokens, and rewrite HTML. The attacks occur when untrusted data enters a web app or dynamic content lacks proper validation. XSS can take various forms, stealing private data, redirecting victims, or performing malicious actions. Three attack categories are stored, reflected, and DOM-based. Developers must implement robust security measures to prevent XSS and safeguard users.

Source: www.cloudflare.com

Click-jacking

Source: https://en.wikipedia.org/

 Clickjacking involves deceiving a user into unknowingly clicking on a link or button different from what they perceive it to be. This malicious technique can lead to various outcomes, such as stealing login credentials or obtaining the user's unwitting consent to install malware. Clickjacking is sometimes referred to as "user interface redressing," although this terminology misrepresents the true meaning of "redress."

Source: developer.mozilla.org/

Cross-site request forgery (CSRF)

Source: https://www.imperva.com/

 Cross-Site Request Forgery (CSRF), also known as XSRF, Sea Surf, or Session Riding, is an attack that deceives a web browser into performing an undesired action within a logged-in user's application. These attacks usually rely on social engineering tactics, like deceptive emails or links, tricking victims into unwittingly sending forged requests to a server. Since the attacker leverages the victim's authenticated session during the attack, distinguishing between legitimate and forged requests becomes nearly impossible.

Source: www.imperva.com/

Denial of service (DoS) & Distributed denial of service (DDoS)

Source: https://privacysavvy.com/

 Attackers employ various methods to overwhelm a specific server or its surrounding infrastructure with diverse forms of attack traffic. As the targeted server becomes incapable of efficiently handling incoming requests, its performance deteriorates, leading to eventual denial of service for legitimate users seeking access. DDoS attacks achieve effectiveness by utilizing multiple compromised computer systems as sources of attack traffic.

Source: www.cloudflare.com

Man-in-the-middle attacks (MITM)

Source: https://wiki.cas.mcmaster.ca

 Man-in-the-middle (MITM) cyber attacks involve security breaches that enable a perpetrator to intercept the exchanged data between two individuals, networks, or computers, earning its name from the attacker inserting themselves between the communicating parties. Essentially, the attacker covertly observes the interaction amid the two entities. This way they may observe and potentially alter sensitive information that is intercepted between parties.

Source: www.fortinet.com

Phishing attacks

Source: https://csoonline.com

 A phishing attack transpires when a malicious individual sends deceptive emails masquerading as legitimate sources, aiming to extract sensitive information from the victim, combining social manipulation and technology. In the execution of such attacks, the perpetrator might dispatch links leading to websites that deceive victims into downloading malware like viruses or revealing private data to the attacker. Frequently, victims remain unaware of the compromise, enabling the attacker to target others within the same organization without raising suspicions.

Source: www.fortinet.com

DNS Spoofing

Source: https://cloudns.net

 The attacker alters DNS records in order to direct internet traffic to a 'spoofed' website.Upon reaching the deceptive website, the target might unwittingly provide confidential details, which the hacker can exploit or sell. Alternatively, the hacker might create a subpar site featuring damaging or provocative material to tarnish a rival company's reputation. In a DNS spoofing attack, the attacker capitalizes on the victim's assumption that the accessed site is genuine.

Source: www.imperva.com

Birthday attack

Source: https://digg.com

 Through a birthday attack, an assailant exploits hash algorithms, utilized to validate message authenticity. These algorithms serve as digital signatures, and the recipient verifies the message's legitimacy by cross-checking it. If a hacker can generate a matching hash to the sender's, they can substitute the original message with their own, deceiving the recipient's device into acceptance. This attack draws its name from the birthday paradox, highlighting that seemingly unique attributes, like birthdays and hashes, can possess unexpected similarities.

Source: www.internetsecurity.tips

Directory / Path traversal

Source: https://wikipedia.org/

 Directory Traversal attacks, also known as Path Traversal attacks, focus on exploiting vulnerabilities in web applications by attempting to navigate beyond the intended directory or folder. The attacker aims to manipulate directory paths within the server's structure in order to traverse upwards in the directory hierarchy. If successful, this type of attack can lead to unauthorized access to sensitive files, including configuration files, databases, and even other websites and files that reside on the same server.

Source: www.tripwire.com
